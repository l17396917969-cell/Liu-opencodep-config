# 灵境航空设计价值观 (Design Values)

**核心定位**：贴合灵境航空工业场景，打造 UX 交互规范与 UI 视觉实现双向协同的标准化技能包，以阶段化迭代、强纪律约束、机器可读格式，保障设计产出高效、统一、可复用，全程贴合航空军工合规、安全、精准的核心诉求。

---

## 5 条核心原则

### 1. 安全可控 (Safety & Control)
- **释义**：所有交互设计必须确保操作安全，防止误操作。在关键业务操作（如军品采购、设备指令下发）中，必须有明确的确认步骤。
- **落地要求**：
  - 关键操作需具备“二次确认”或“延时撤回”机制。
  - 重要状态变更必须实时反馈，严禁出现“操作后无响应”的黑盒状态。
  - 系统错误需提供明确的退出路径和补救方案。

### 2. 精准高效 (Precision & Efficiency)
- **释义**：复杂功能简单化表达，避免过度设计。设计应聚焦于任务本身，减少干扰，确保信息传递的精确性。
- **落地要求**：
  - 指标展示必须符合航空工业精度要求，避免模糊用语。
  - 高频操作路径（如审批、入库、派单）点击次数控制在 3 次以内。
  - 自动过滤低价值信息，突出关键告警和待办项。

### 3. 合规可溯 (Compliance & Traceability)
- **释义**：全流程留存可追溯记录，适配 FAA/EASA 等行业监管要求。所有操作、权限变更、异常处理均需有据可查。
- **落地要求**：
  - 所有关键交互模式内置“审计追踪”能力。
  - 任何权限受限的操作必须告知原因及申请路径，不可直接“消失”。
  - 流程节点需标注合规卡点，确保不合规操作无法通过。

### 4. 极简交互 (Minimalist Interaction)
- **释义**：在复杂航空场景下，保持界面的极简性，降低用户的认知负担和操作复杂度。
- **落地要求**：
  - 遵循“按需显示”原则，默认展示核心指标，更多维度通过钻取或联动展示。
  - 组件交互逻辑统一（如：所有“确定”按钮在右侧，所有“返回”在左上角）。
  - 避免冗余的装饰性交互，优先保障功能流畅度。

### 5. 虚实融合 (Physical-Digital Fusion)
- **释义**：数字世界与物理世界信息无缝衔接。通过数字孪生和虚实同步，提升对现场态势的感知力。
- **落地要求**：
  - 2D UI 叠加层应与 3D 场景中的空间锚点保持视觉一致性。
  - 实时数据刷新频率需匹配业务场景（如：运控场景刷新间隔 ≤ 60s）。
  - 支持从虚拟场景快速定位到物理实体（如：点击设备标签跳转至监控明细）。
